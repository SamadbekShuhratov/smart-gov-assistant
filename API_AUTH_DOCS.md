# Authentication API Endpoints\n\n## Base URL\n```\nhttp://localhost:8000\n```\n\n## Endpoints\n\n### 1. POST /login\nUser login endpoint\n\n**Request:**\n```json\n{\n  \"username\": \"demo\",\n  \"password\": \"demo123\"\n}\n```\n\n**Response (200 OK):**\n```json\n{\n  \"access_token\": \"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...\",\n  \"token_type\": \"bearer\"\n}\n```\n\n**Error (401):**\n```json\n{\n  \"detail\": \"Invalid credentials\"\n}\n```\n\n**Frontend Usage:**\n```javascript\nconst response = await loginUser({ username, password });\nsetAuthToken(response.access_token);\nsetAuthUsername(username);\nlocalStorage.setItem(\"authToken\", response.access_token);\nlocalStorage.setItem(\"authUsername\", username);\n```\n\n---\n\n### 2. POST /register\nUser registration endpoint\n\n**Request:**\n```json\n{\n  \"full_name\": \"Aziz Karim\",\n  \"passport_number\": \"AA123456\",\n  \"birth_date\": \"1990-05-15\",\n  \"address\": \"Tashkent, Uzbekistan\",\n  \"username\": \"azizkarim\",\n  \"password\": \"secure123\",\n  \"family_members\": [\n    {\n      \"name\": \"Family Member Name\",\n      \"birth_date\": \"2020-01-01\"\n    }\n  ]\n}\n```\n\n**Response (200 OK):**\n```json\n{\n  \"username\": \"azizkarim\",\n  \"message\": \"User registered successfully\"\n}\n```\n\n**Error (400):**\n```json\n{\n  \"detail\": \"Username already exists\"\n}\n```\n\n**Frontend Usage:**\n```javascript\nconst response = await registerUser({\n  full_name, passport_number, birth_date, address,\n  username, password, family_members\n});\n// Then user can login with new credentials\n```\n\n---\n\n### 3. GET /profile\nGet authenticated user profile (requires token)\n\n**Headers:**\n```\nAuthorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...\n```\n\n**Response (200 OK):**\n```json\n{\n  \"username\": \"demo\",\n  \"full_name\": \"Demo User\",\n  \"passport_number\": \"AA000000\",\n  \"birth_date\": \"1990-01-01\",\n  \"address\": \"Demo Address\"\n}\n```\n\n**Error (401):**\n```json\n{\n  \"detail\": \"Invalid or expired token\"\n}\n```\n\n---\n\n## Authentication Flow\n\n### Flow Diagram\n```\n1. User visits app (autToken empty)\n   ↓\n2. Frontend shows auth modal\n   ├─ User clicks Login → POST /login\n   ├─ Backend validates credentials\n   ├─ Returns access_token\n   ↓\n3. Frontend stores token in:\n   ├─ React state (authToken)\n   ├─ localStorage (persistence)\n   └─ All future requests include token in header\n   ↓\n4. User can now:\n   ├─ Access main app\n   ├─ Fetch user profile (uses /profile endpoint)\n   ├─ Request autofill data (uses token)\n   └─ Search services\n   ↓\n5. On page reload:\n   ├─ Frontend reads localStorage\n   ├─ Restores authToken\n   ├─ User already logged in (no login required)\n   ↓\n6. User clicks Logout:\n   ├─ Frontend clears state\n   ├─ localStorage cleared\n   ├─ Auth modal reappears\n```\n\n---\n\n## Token Usage in Frontend\n\n### Adding Token to API Requests\n```javascript\n// All API calls that need auth should include token:\n\nconst headers = {\n  \"Content-Type\": \"application/json\",\n  \"Authorization\": `Bearer ${authToken}`  // ← Include token\n};\n\nfetch(\"/autofill?username=demo\", {\n  headers\n});\n```\n\n### Example: fetchAutofillData with Token\n```javascript\nexport async function fetchAutofillData(username, formType = \"full_application_all\") {\n  const params = new URLSearchParams({\n    username,\n    form_type: formType,\n  });\n\n  const response = await fetch(\n    `${API_BASE}/autofill?${params.toString()}`,\n    {\n      headers: {\n        \"Authorization\": `Bearer ${authToken}`  // Token needed\n      }\n    }\n  );\n\n  if (!response.ok) {\n    throw new Error(`Autofill request failed: ${response.status}`);\n  }\n\n  return response.json();\n}\n```\n\n---\n\n## Demo Credentials\n\n### Pre-configured User\n```\nUsername: demo\nPassword: Unauthorized     → Missing/invalid token\n404 Not Found        → Endpoint doesn't exist\n500 Server Error      → Backend error\n```\n\n### Frontend Error Handling\n```javascript\ntry {\n  const response = await loginUser({ username, password });\n  setAuthToken(response.access_token);\n} catch (error) {\n  // Show error toast\n  setAuthError(\"Login failed. Please check credentials.\");\n}\n```\n\n---\n\n## CORS Configuration\n\nThe backend is configured to accept requests from:\n- Local development: `http://localhost:5173`, `http://localhost:5174`, `http://localhost:5175`\n- Local network: `http://10.0.1.128:5175`\n\n---\n\n## Testing with cURL\n\n### Login Test\n```bash\ncurl -X POST http://localhost:8000/login \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\"username\": \"demo\", \"password\": \"demo123\"}'\n\n# Response:\n# {\"access_token\": \"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...\", \"token_type\": \"bearer\"}\n```\n\n### Register Test\n```bash\ncurl -X POST http://localhost:8000/register \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"full_name\": \"New User\",\n    \"passport_number\": \"AA123456\",\n    \"birth_date\": \"1990-01-01\",\n    \"address\": \"Tashkent\",\n    \"username\": \"newuser\",\n    \"password\": \"pass123\",\n    \"family_members\": []\n  }'\n\n# Response:\n# {\"username\": \"newuser\", \"message\": \"User registered successfully\"}\n```\n\n### Profile Test (requires token)\n```bash\nTOKEN=\"<access_token_from_login_response>\"\n\ncurl -X GET http://localhost:8000/profile \\\n  -H \"Authorization: Bearer $TOKEN\"\n\n# Response:\n# {\"username\": \"demo\", \"full_name\": \"Demo User\", ...}\n```\n\n---\n\n## API Documentation\n\nInteractive API docs available at:\n- **Swagger UI**: http://localhost:8000/docs\n- **ReDoc**: http://localhost:8000/redoc\n"
